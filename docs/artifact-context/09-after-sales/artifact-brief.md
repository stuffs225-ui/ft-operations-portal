# Module 09 — After Sales

## Accounts with Access
`afs`, `coo`, `ops`, `admin`, `stuffs`

## Routes Captured

| Route | Name |
|-------|------|
| `/after-sales` | After Sales Dashboard |
| `/after-sales/maintenance` | Maintenance Requests |
| `/after-sales/maintenance/new` | New Maintenance Request |

## Review Checklist

- [ ] After-sales dashboard has open requests count and average resolution time
- [ ] Maintenance request list shows priority and age clearly
- [ ] New maintenance request form has link-to-project field
- [ ] Overdue requests are visually flagged
- [ ] Status transitions are clear (open → in-progress → resolved)

## Artifact Instructions

After-sales requests are post-delivery — link to project is optional but important to surface.
Priority must be visually distinct (urgent vs. standard vs. low).
