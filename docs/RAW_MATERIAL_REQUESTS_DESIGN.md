# Raw Material Requests Design

## Overview
Raw Material Requests (RMRs) are created by the Factory team to request materials from Procurement. They can be project-related (linked to a specific project, WO, and vehicle line) or stock requests (general workshop consumables).

## Request Number Format
`RMR-YYYY-NNNN` — auto-generated by database trigger.

## Request Types

| Type | Description |
|---|---|
| project_related | Linked to an approved Saudi project + WO. Can reference a specific vehicle/item line. |
| stock | General stock replenishment. No project linkage required. |

## RMR Lifecycle

```
draft → submitted → under_review → sent_to_procurement → partially_fulfilled → fulfilled
                                  ↘ rejected
                                  ↘ cancelled
```

| Status | Meaning |
|---|---|
| draft | Created but not yet submitted |
| submitted | Submitted by Factory for review |
| under_review | Being reviewed by Factory Lead / Ops |
| sent_to_procurement | Forwarded to Procurement team |
| partially_fulfilled | Some items received from Procurement |
| fulfilled | All items received |
| rejected | Request rejected |
| cancelled | Cancelled by requestor |

## File Upload

In Phase 6, file upload stores metadata only:
- `file_name`: name of the uploaded file
- `file_type`: excel_bom, excel_boq, pdf_specification, other
- `storage_path`: null in dev mode; Supabase Storage path in production
- `parsing_status`: not_parsed | pending_future_parser | parsed | failed

## Who Can Create RMRs

- factory_user, admin, operations_manager

## Who Can View RMRs

- factory_user: all RMRs they created
- admin, operations_manager: all RMRs
- procurement_user: RMRs with status=sent_to_procurement
- store_user: all RMRs (read-only, for receiving preparation)
- viewer: read-only
